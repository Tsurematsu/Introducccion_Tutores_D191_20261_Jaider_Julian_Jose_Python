# Commit Tags

| Etiqueta | Cuándo usarla |
|---|---|
| `(add)` | Archivos nuevos que no son código funcional: docs, configs, assets, md |
| `(feature)` | Nueva funcionalidad completa o inicio formal de una |
| `(update)` | Mejora o extensión de una funcionalidad ya existente |
| `(fix)` | Corrección de un bug o comportamiento incorrecto |
| `(refactor)` | Reorganización de código sin cambiar comportamiento |
| `(breaking)` | Cambio estructural que afecta otras partes del sistema |
| `(wip)` | Trabajo en progreso, commit intermedio que no está terminado |
